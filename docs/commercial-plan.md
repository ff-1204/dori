> 전략 요약: **커스텀 도메인 + 상업 허용 호스팅 이전 → 법적 준비(처리방침·CMP) → AdSense/H5 Games Ads 승인 → 절제된 광고 활성화.**

### 콘텐츠 셀프 점검 결과 (2026-07-18)

**통과 — 정책 금지 콘텐츠 기준 문제 없음**

- **도박**: 실베팅·구매 유도·사업자 링크·당첨 보장 표현 전무. 로또는 순수 번호 생성기 — "모든 조합의 확률은 같아요" 상시 고지, 카운트다운 없음. 오락 목적 고지 3중(LICENSE·README·noscript).
- **폭력·성인**: 러시안 룰렛은 총기 묘사 없음(추상 원판 + 점 6개, "빵!"·"찰칵" 텍스트, 벌칙 당첨 유쾌 프레이밍) — 충격적 콘텐츠 정책 무관. 성인·주류·약물·혐오 없음.
- **크롤 설정**: robots.txt는 검색봇 전체 Allow(AdSense 크롤러 Mediapartners-Google 포함), noai 메타는 색인과 무관 — 심사 크롤에 지장 없음.
- **기만 요소**: 없음(정직 매핑·다크 패턴 금지가 설계 원칙).

**보완 필요 — 콘텐츠 충분성이 최대 탈락 리스크**

1. **크롤러/심사관이 보는 HTML이 사실상 비어 있음** — 단일 페이지 + 전체 캔버스 구조라 실제 텍스트는 noscript 2문단뿐. 게임 10종의 볼륨이 HTML에 전혀 드러나지 않아 **"낮은 가치의 콘텐츠(최소 콘텐츠)" 반려의 전형적 패턴**. → 대응: 크롤 가능한 정적 페이지 세트 추가(소개/게임별 안내 — README·docs에 원고 이미 있음), 허브/하단에서 실제 `<a>` 링크로 연결하고 sitemap.xml에 등록. 해시 딥링크(#roulette)는 개별 색인이 안 되므로 대체가 아니라 보완이 필요. **5단계에 항목 추가**(privacy.html과 한 묶음으로 작업).
2. **운영자 정보 부재** — 사이트 내 소개(About)·연락 수단(이메일)이 없음. 신뢰 신호로 정적 페이지에 포함 권장.
3. **로또 게임**: 정책상 문제 없으나 심사관 편차 리스크는 잔존 — 반려 사유로 지목되면 허브 `ready` 플래그로 비노출 처리(기존 대응 유지).

## 2단계. 도메인 (연 1–3만원 — 전체 경로에서 유일한 비용)

**도메인 구매는 피할 수 없는 첫 단추다(2026-07-18 확정).** github.io 그대로는 스크립트를 붙일 수는 있어도
**AdSense 사이트 등록 단계에서 거부**된다(루트 도메인 github.io가 GitHub 소유 — 심사 시작 자체가 불가).
무료 대안도 전부 막혀 있다: 무료 서브도메인(eu.org 등)은 같은 소유권 문제, 무료 TLD(.tk 계열)는 서비스 붕괴·스팸 취급.

- **후보 TLD**: `.com`(1.5–2만원/년, 무난·신뢰) / `.games` `.fun`(2–4만원, 정체성) / `.kr`(2만원 안팎, 국내). 예: `dori.games`, `playdori.com`
- **등록처**: Cloudflare Registrar(도매가 그대로 — 최저가, 해외카드 필요), 가비아·호스팅케이알(국내 결제 편의)
- **유의**: 첫해 파격 할인은 **갱신가** 확인. WHOIS 개인정보 보호 옵션(대부분 무료) 켜기. 상표 검색(1단계)과 도메인명 충돌 확인
- 담당: **사용자**(구매) → 구매 후 도메인명을 알려주면 다음 단계부터 Claude가 이어서 진행 | 상태 ✅ 2026-07-18 — **`dori.io.kr` 구매 완료**

## 3단계. 호스팅 — 옵션 A(최소 비용 시작) 또는 옵션 B(정석)

### 옵션 A: GitHub Pages 유지 + 커스텀 도메인 연결 (추가 비용 0 — 시작용 권장)

호스팅 이전 없이 도메인만 연결해 AdSense 등록 요건을 충족한다. GitHub Pages는 커스텀 도메인을 공식 지원한다
(https://docs.github.com/pages/configuring-a-custom-domain-for-your-github-pages-site).

1. 저장소 **Settings → Pages → Custom domain**에 도메인 입력 → 저장(루트에 `CNAME` 파일 자동 커밋 — **이후 커밋에서 건드리지 말 것**)
2. 도메인 등록처 DNS 설정:
   - 루트(apex) `dori.games` → **A 레코드 4개**: `185.199.108.153` `185.199.109.153` `185.199.110.153` `185.199.111.153`
   - `www` 서브도메인 → **CNAME**: `ff-1204.github.io`
   - 루트+www 둘 다 걸고 한쪽을 기본으로(나머지는 GitHub이 리다이렉트)
3. DNS 전파 후 **Enforce HTTPS** 체크(Let's Encrypt 자동 발급)
4. 계정 Settings → Pages에서 **TXT 레코드 도메인 검증**(도메인 탈취 방지 — 권장)
- **남는 리스크**: GitHub Pages 약관의 상업 운영 제한은 회색지대로 유지 — 수익이 유의미해지면 옵션 B로 이전
- 담당: 사용자(설정·DNS) + Claude(검증·코드 정합) | 상태 ✅ 2026-07-18 — DNS(A 4개) 반영·https 200·http→https 301·github.io→새 도메인 301·**TXT 도메인 검증 통과**까지 전부 완료

### 옵션 B: Cloudflare Pages 이전 (무료 — 수익화 본격화 시)

정적 사이트라 빌드 설정 없이 그대로 올라간다. 약관 리스크가 없어지는 정석 경로.

1. Cloudflare 계정 생성 → **Workers & Pages → Pages → Connect to Git** → `ff-1204/dori` 연결
2. 빌드 설정: 프레임워크 None, 빌드 명령 없음, 출력 디렉터리 `/` (루트)
3. 배포 확인(`*.pages.dev` 주소) → **Custom domains**에 도메인 연결(DNS 자동 구성), HTTPS 자동
4. 전환기: GitHub Pages 병행 유지(guard가 양쪽 허용) 후 정리
- 담당: 사용자(계정·연결 클릭) + Claude(절차 안내·검증) | 상태 ☐

> 권장 진행: **옵션 A로 시작 → AdSense 승인·수익 확인 → 필요 시 옵션 B 이전.** 도메인은 그대로 옮겨 연결하면 되므로 전환 비용이 거의 없다.
