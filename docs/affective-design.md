# 감성적 설계 — 심리·생리 패턴

게임 요소와 색 배치를 정할 때 **사람의 심리적 패턴**과 **생리적(시간 기반) 패턴**을 근거로 삼아
"감성적으로" 만드는 지침. 이 문서는 dori가 왜 그 색·그 연출·그 분위기를 쓰는지의 **근거**를 제공한다.

> **상위 원칙**: [design-principles.md](./design-principles.md)(게이미피케이션·어포던스, **정직함**)를 우선한다.
> 감성 연출은 분위기만 담당하며, **게임 정보(확률·결과·조작)를 왜곡하면 안 된다.**
> 색 값·간격·이징은 [visual-polish.md](./visual-polish.md)와 함께 본다.

---

## 1. 심리적 패턴 (Psychological)

### 1-1. 색채 심리

- **온도**: 난색(빨강·주황·노랑)은 각성↑·식욕↑·주목, 한색(파랑·청록)은 진정·신뢰·집중.
- **채도·명도**: 채도/명도가 높을수록 **각성(arousal)↑**. 차분함을 원하면 채도를 낮춘다.
- **색상 연상**(참고, 문화 의존):

| 색 | 대표 연상 | dori 용도 |
|----|-----------|-----------|
| 빨강 | 긴장·경고·식욕·열정 | 위험/꽝/패배 |
| 주황 | 활기·식욕·따뜻함 | 시간대(오후) 분위기 |
| 노랑 | 주의·활력·즐거움 | 경고/강조 포인트 |
| 초록 | 안전·성공·안정 | 당첨/성공 |
| 파랑 | 신뢰·차분·집중 | 주요 강조(Primary) |
| 보라 | 신비·프리미엄 | 저녁 분위기 |

- **접근성**: 색만으로 의미를 전달하지 않는다(색각 이상). **색 + 아이콘/라벨/명도 대비** 병행.
- **dori 규칙**: **의미색은 고정**(성공=초록, 위험=빨강, 강조=파랑), 감정 톤은 §2·생리(시간) 오버레이로 얹는다.

### 1-2. 감정 모델

- **러셀 원형 모델(Circumplex)**: 감정을 **정서가(valence: 긍정↔부정) × 각성(arousal: 높음↔낮음)** 2축으로 본다.
- **플루칙 감정 바퀴**: 기대·기쁨·놀람·공포·신뢰 등 기본 감정과 강도.
- **게임별 목표 감정 좌표(dori)**:

| 게임/순간 | 목표 감정 | 좌표(가치·각성) |
|-----------|-----------|-----------------|
| 룰렛 돌리는 중 | 설렘·기대 | 긍정 · 고각성 |
| 복불복 진행 중 | 스릴·긴장 | 중립 · 고각성 |
| 당첨/성공 | 환희 | 긍정 · 고각성 |
| 꽝/패배 | 아쉬움(가볍게) | 약부정 · 중각성 |
| 결과 후 여운 | 안도·만족 | 긍정 · 저각성 |

→ 색·모션·사운드·이징을 이 좌표에 맞춰 조율한다(고각성=빠른 모션·높은 채도·상승 피치).

### 1-3. 감성공학 (Kansei Engineering)

감성어(설렘·귀여움·긴장·경쾌)를 **디자인 파라미터**로 번역한다.

| 감성어 | 색 | 모션/이징 | 형태 |
|--------|----|-----------| -----|
| 설렘·경쾌 | 밝은 난색·시안 | `Back.easeOut`, 통통 | 둥근 모서리 |
| 긴장·스릴 | 저채도+포인트 빨강 | 점점 빠르게, 화면 조임 | 각진 대비 |
| 안정·차분 | 저채도 한색 | 느린 `Quad.easeInOut` | 넓은 여백 |

### 1-4. 기대·기억의 심리

- **절정-대미 법칙(Peak-End Rule)**: 경험의 기억은 **절정 순간 + 마지막 순간**이 지배. → **결과 공개 연출**과 **엔딩(다시 하기 유도)**에 마감 예산을 집중.
- **자이가르닉 효과**: 미완의 긴장이 기억에 남는다 → 결과 직전의 텀(감속 마지막)이 몰입을 만든다.
- **손실 회피(loss aversion)**: 잃는 것이 얻는 것보다 크게 느껴짐 → 복불복 "걸리면 진다"의 긴장 근거(단, 벌칙은 가볍고 유쾌하게).
- **기대감(anticipation)**: 결과보다 **과정의 빌드업**이 재미([game-theory.md](./game-theory.md) §5·§8).

### 1-5. 인지·의사결정

- **힉의 법칙(Hick's Law)**: 선택지가 많을수록 결정이 느려짐 → 허브를 **범주(결정·뽑기·복불복)**로 묶어 인지 부하↓.
- (Fitts의 법칙·게슈탈트·시각 위계는 [responsive-design.md](./responsive-design.md)·[visual-polish.md](./visual-polish.md) 참조.)

## 2. 생리적 패턴 (Physiological) — 시간 기반

사람의 몸은 시간에 따라 각성·기분·빛 민감도가 바뀐다. dori는 **접속 시각**으로 이를 추정해 분위기에 반영한다.

### 2-1. 서카디안 리듬(Circadian Rhythm)

- 하루 주기의 각성/기분 곡선: 아침 각성 상승 → **점심 무렵 피크** → **오후 초 식곤증(post-lunch dip)** → 저녁 하강 → 밤 저각성(수면 준비).

### 2-2. 시간대별 특성

| 시간대 | 시각 | 생리 상태 | 무드 톤(글로우) | 인사말(예) |
|--------|------|-----------|-----------------|------------|
| 아침 | 05–11 | 각성 상승 | 연한 앰버(상쾌) | "상쾌한 아침이에요" |
| 점심 | 11–14 | 각성 피크 | 밝은 시안(활기) | "점심시간! 뭐 먹을까요?" |
| 오후 | 14–17 | 식곤증 dip | 웜 오렌지(포근) | "나른한 오후네요" |
| 저녁 | 17–21 | 각성 하강 | 퍼플/노을(차분) | "수고한 저녁이에요" |
| 밤 | 21–05 | 저각성·수면 준비 | 딤 웜·저휘도 | "고요한 밤이에요" |

### 2-3. 색온도와 빛

- 자연광은 낮에 쿨, 저녁·밤에 웜. 화면도 이를 따르면 자연스럽다.
- **블루라이트**는 밤에 멜라토닌 분비를 억제(각성). → **밤에는 파랑을 줄이고 웜·저휘도**로(눈 피로·수면 방해↓). *(그래서 dori의 밤 테마는 흔한 "네온 블루"가 아니라 딤 웜이다.)*

### 2-4. 울트라디안 리듬(≈90분)

- 집중은 약 90분 주기로 오르내린다 → 세션은 **짧게**, 강요 없는 휴식. dori 미니게임은 본래 짧아 부합.

### 2-5. dori 적용: 시간대 적응형 분위기

- 접속 시각 → **phase(아침/점심/오후/저녁/밤)** → 배경 글로우·인사말·톤을 바꾼다.
- 구현: `src/timeOfDay.js`의 `getTimePhase()`·`applyTimeAtmosphere(scene)`. (`new Date()`는 브라우저 런타임에서 정상 동작.)
- **정직 가드레일**: 시간 테마는 **배경 분위기만**. 확률·결과·버튼 등 **정보를 전달하는 색은 시간과 무관하게 고정**한다(어포던스 정직한 매핑).

## 3. dori 적용 규칙 요약

- **의미색 고정**(성공 초록 / 위험 빨강 / 강조 파랑) + **시간 글로우 오버레이**(저알파, 정보 아님).
- 게임/순간별 **목표 감정 좌표**(§1-2)에 맞춰 색·모션·사운드·이징을 조율.
- **절정(결과 공개)과 엔딩**에 연출을 집중(Peak-End).
- **밤엔 파랑↓·웜·저휘도**(블루라이트·수면 배려).
- 허브는 **범주화**로 선택 부하↓(Hick).
- 감성 연출이 **정직한 정보 전달을 해치지 않게**([design-principles.md](./design-principles.md)).

## 4. 구현 메모

- 시간 글로우 색은 **분위기 전용**이라 UI 팔레트의 예외를 허용하되 **저알파(0.08–0.14)**로 절제 — 정보색(성공/위험/강조)과 혼동되지 않게 한다.
- 새 미니게임도 `MiniGame` 베이스를 상속하면 시간 분위기가 자동 적용된다.

## 5. 체크리스트

- [ ] 정보색(성공/위험/강조)이 시간과 무관하게 일정한가(정직)
- [ ] 결과 공개·엔딩 순간에 연출이 집중됐는가(Peak-End)
- [ ] 밤 테마가 저자극·웜인가(블루라이트 배려)
- [ ] 게임의 목표 감정과 색·모션·사운드가 일치하는가
- [ ] 선택지가 범주로 정리돼 인지 부하가 낮은가
