# 상업화 규칙 (Commercialization) — 광고·수익화 대비

상업적으로 배포·수익화(광고 포함)할 수 있도록 지키는 규칙.

> **상위 원칙**: [design-principles.md](./design-principles.md)(게이미피케이션·어포던스, **정직함**)를 우선한다.
> 수익화도 **윤리 가드레일 안에서** — 다크 패턴·오클릭 유도·기만 광고 금지.

---

## 0. 호스팅 주의 (중요)

- **GitHub Pages**는 주로 상업적 목적(온라인 비즈니스 운영)의 호스팅을 **약관상 제한**한다.
  광고로 본격 수익화하면 ToS 위반 소지가 있다.
- **권장**: 상업화 단계에서 **Netlify / Vercel / Cloudflare Pages** + 커스텀 도메인으로 이전.
  정적 사이트라 이전 비용이 거의 없다(개발 중에는 GitHub Pages 유지 OK).

## ⭐ 개발 중 상시 준수 수칙 (Dev-time compliance)

**핵심 원칙**: 상업 안전은 "릴리스 때 몰아서 점검"이 아니라 **추가하는 순간 검증·기록**한다.
아래는 개발하는 내내 지키는 실무 수칙이다.

### 무엇이든 추가하기 전에 (라이브러리·폰트·이미지·사운드·코드 스니펫)
1. **라이선스부터 확인** → 상업적 사용이 허용되는가?
2. **허용**: MIT · Apache-2.0 · BSD · ISC · Zlib · **CC0** · **OFL**(폰트) · CC-BY(출처표기 조건).
3. **금지/주의**:
   - **GPL / AGPL / LGPL** — 카피레프트 전염(소스 공개 강제) 위험.
   - **CC BY-NC** — 비상업 전용(상업 불가).
   - **CC BY-SA** — 동일조건 변경 허락(전염).
   - "**개인용 무료**", "무료 체험", 저작권 불명 에셋.
4. **즉시 [licenses.md](./licenses.md)에 기록**(항목·버전·라이선스·출처 URL). 나중으로 미루지 않는다.
5. 원본이 라이선스 파일 동봉을 요구하면(OFL 등) 함께 보관.

### 코드 스니펫 주의
- **Stack Overflow 코드 = CC BY-SA** → 그대로 붙여넣기 주의(출처표기·전염 소지). 이해 후 **직접 작성** 권장.
- 튜토리얼/예제 코드도 라이선스 확인. Phaser 공식 예제는 라이선스 확인 후 사용.

### 자산 (이미지·폰트·사운드)
- 구글 이미지·유튜브 음원 등을 **그냥 가져오지 않는다.**
- 출처는 **자체 제작 · CC0 · 상업 라이선스 구매분**만. 구매 영수증/출처 URL 보관.
- **AI 생성 자산**: 생성 도구의 **상업 사용 약관**을 확인(모델·플랜별로 상이). 확인 결과를 licenses.md에 메모.

### 브랜드·IP·상표
- 실제 **상표/브랜드명·로고·캐릭터·저작 음악** 사용 금지.
- 메뉴 등은 **일반명**만(짜장면·초밥 OK / 특정 브랜드명 지양).
- 프로젝트·게임 이름이 기존 상표와 충돌하지 않는지 확인.

### 개인정보·트래킹
- 개발 중이라도 **PII 수집·쿠키를 설정하는 서드파티 스크립트**를 무단 삽입하지 않는다.
- 분석/광고 SDK는 개인정보처리방침·동의(CMP) 전까지 **비활성 유지**(현재 `ads.js` 비활성).

### Git 위생
- 저작권 자산을 **잠깐이라도 커밋하지 않는다**(히스토리에 영구 잔존). 실수 시 히스토리 정리가 필요해진다.

### 커밋 전 3문 점검
- [ ] 새로 넣은 서드파티가 **상업 허용 라이선스**인가?
- [ ] **licenses.md에 기록**했는가?
- [ ] **브랜드·저작권 자산을 무단 포함**하지 않았는가?

## 1. 라이선스 규칙 (상업적 사용 가능한 것만)

- **코드/라이브러리**: 상업 허용 라이선스(MIT · Apache-2.0 · BSD · ISC)만 도입.
  - Phaser 3 = **MIT** → 상업 사용 OK.
- **폰트**: 상업 허용만(예: Pretendard·Noto Sans KR = OFL). 무단 폰트 금지.
- **이미지/사운드/아트**: **CC0** 또는 **상업 라이선스 구매분**만. 저작권 불명·"개인용 무료" 에셋 금지.
- **상표/IP**: 브랜드 로고·캐릭터·게임명(예: 실제 상품명) 무단 사용 금지.
- **게임 명칭 유의(검토 완료 2026-07)**: 게임 **메커니즘**(통에 칼 꽂기·이빨 누르기 등)은 저작권 보호 대상이 아니어서 자체 구현은 문제없음. 다만 **'통아저씨'·'악어 이빨' 같은 통용 명칭이 국내 상표로 등록돼 있을 가능성** 있음 → **수익화 개시 전 상표 검색(KIPRIS) 필수**, 충돌 시 대체명(예: '해적통', '악어 치과') 전환 — 코드상 씬 키와 표기 라벨이 분리돼 있어 전환 비용 낮음.
- **기록 의무**: 도입한 모든 서드파티의 출처·라이선스·URL을 [licenses.md](./licenses.md)에 남기고 릴리스 전 점검.

## 2. 개인정보·법적 준수 (광고의 전제)

- 광고(특히 **맞춤형**)는 쿠키/광고 식별자를 쓰므로 **동의**가 필요.
- **개인정보처리방침(Privacy Policy)** 페이지 필수.
- 지역 규제: **GDPR**(EU)·**CCPA**(미 캘리포니아)·**개인정보보호법**(국내) → **동의 배너(CMP)** 도입.
- **아동 대상 여부**: COPPA/연령 정책. 아동 트래픽이 있으면 **비맞춤형(non-personalized) 광고**로.
- **도박 오인 방지**: 룰렛·러시안 룰렛은 재미용이며 실제 베팅이 아님을 명확히. 광고 카테고리에서 **실제 도박/베팅 광고 차단**.
- **사용 제한 고지**: 오락 목적 전용 — 금전이 걸린 도박·사행·기만 등 **비윤리적 목적 사용 금지**를 LICENSE(사용 제한 조항)·README·사이트(noscript)에 명시. 서비스가 도박 플랫폼으로 쓰이는 정황 발견 시 차단 조치.
- **게임물 등급분류(국내)**: 게임산업진흥법상 '게임물' 유통 시 등급분류 의무가 적용될 수 있음. 무료·비수익 웹 미니게임은 실무상 저위험이나, **광고 수익화 시점에 자체등급분류 대상 여부/면제 요건을 확인**한다(체크리스트 반영).
- **개인정보 현황(2026-07 검토)**: 수집 0 — 분석 도구·쿠키 없음, localStorage는 기기 내 설정 저장 전용, 단청 입력은 클라이언트에서만 소비(전송 없음) → 현재는 처리방침 불요, **광고 도입 전 필수**(§2).

## 3. 광고 설계 규칙 (수익과 UX 양립)

- **포맷**: 배너 / 전면(interstitial) / 보상형(rewarded).
- **배치 원칙**:
  - **플레이 중 삽입 절대 금지**(룰렛 도는 중, 복불복 진행 중).
  - 자연스러운 **경계에서만**: 결과 화면 이후, 허브 복귀 시.
  - **전면광고 빈도 제한**: 예) N판마다 1회 + 최소 간격.
  - **보상형은 자발적만**: "광고 보고 한 번 더 돌리기"처럼 유저가 선택. 강제 금지.
- **오클릭 방지**(어포던스/정직): 광고를 게임 버튼 옆에 붙이지 않기, **닫기(X)는 크고 명확**, 로딩 직후 즉시 클릭 유도 금지.
- **다크 패턴 금지**: 가짜 X, 강제 시청, 오해 유도 → [design-principles.md](./design-principles.md) 위반.
- **성능/견고성**: 광고 SDK는 **지연 로드**, 게임 로딩을 막지 않으며, **광고 실패/차단 시에도 게임은 정상 진행**.

## 4. 아키텍처 대비 (지금 심어두는 것)

- **광고 슬롯 추상화**: `src/ads.js`에 `maybeShowInterstitial()` · `showRewarded()` **no-op 스텁**을 두었다.
  실제 SDK는 나중에 이 안에서 연결(게임 코드는 손 안 대고 교체 가능).
- **연결 지점**: 게임 종료(결과 후) · 허브 복귀 시점에서만 호출(현재는 미연결).
- **일시정지 여지**: 광고 표시 중 게임이 멈추도록 `MiniGame`에 pause 훅을 추가할 여지를 남긴다.
- **설정 분리**: `AdConfig`(enabled·빈도)로 on/off·튜닝.

## 4-1. Google 광고 연결 계획

| 제품 | 적합성 | 용도 |
|------|--------|------|
| **AdSense (디스플레이)** | ✅ 웹페이지 배너 | 허브 등 비플레이 화면 |
| **H5 Games Ads** (AdSense/Ad Manager) | ✅ **HTML5 게임용** | 전면(interstitial)·**보상형(rewarded)** |
| **AdMob** | ❌ | 모바일 **앱 전용**(웹 게임엔 부적합) |

**전제 조건**
- **커스텀 도메인 필요**: `github.io` 서브도메인은 AdSense 승인이 어렵다(도메인 소유 확인 + GitHub Pages 상업 약관) → §0대로 이전.
- **개인정보처리방침 + 동의(CMP)**: EEA/영국 트래픽엔 **구글 인증 CMP** 필수(§2).
- **승인 심사**: 콘텐츠·정책 통과.

**준비해 둔 것(현재 미연결·비활성)**
- `src/ads.js`: **H5 Games Ads `adBreak` API 형태**의 어댑터(`maybeShowInterstitial`·`showRewarded`). `AdConfig.enabled=false`, `provider='none'`으로 꺼져 있음.
- `index.html`: H5 Games Ads 스크립트를 **주석 처리**로 넣어둠(로드 안 됨).

**활성화 절차(나중에)**
1. 커스텀 도메인 + 상업 허용 호스팅 이전, AdSense 승인, CMP 구비.
2. `index.html`의 H5 Games Ads 스크립트 주석 해제 + `ca-pub-XXXX`를 실제 퍼블리셔 ID로 교체.
3. `ads.js`의 `AdConfig.enabled=true`, `provider='google-h5'`, `client` 설정.
4. 게임 종료/허브 복귀 지점에서 `maybeShowInterstitial()` 호출 배선(게임 pause/resume 연결).

## 5. 수익화 모델 (옵션)

- 광고(배너/전면/보상형) · **광고 제거 IAP** · 프리미엄 게임팩.
- MVP 권장: **보상형 + 절제된 전면**(플레이를 해치지 않는 선에서).

## 5-1. 무단 복제·재호스팅 방지

정적 게임은 소스가 공개되어 **완벽 차단은 불가**하다. 계층적으로 억제한다.

| 계층 | 수단 | 강도 | 상태 |
|------|------|------|------|
| 법적 | **독점 라이선스**(루트 `LICENSE`, All Rights Reserved) | 근거 확보 | ✅ 적용 |
| 실행 | **도메인 잠금 가드**(`src/guard.js`) — 허용 호스트 아니면 차단 | 중(우회 가능) | ✅ 적용 |
| 수익 | **AdSense 도메인 허용목록 + `ads.txt`** — 클론이 내 pub-id로 수익 못 냄 | 강 | ⏳ 광고 연결 시 |
| 난이도 | 코드 압축/난독화 | 약(억지) | ⏳ 선택 |

- **도메인 잠금**: `guard.js`의 `ALLOWED_HOSTS`에 등록된 호스트에서만 게임 시작. 커스텀 도메인 이전 시 목록에 추가한다.
- **주의(중요)**: 무료 GitHub Pages는 **public 저장소**라 소스가 그대로 노출된다. 소스 은닉이 필요하면 **private 저장소 + Pages(유료)** 또는 상업 호스팅으로 이전한다.
- 핵심 위협(광고 수익 도용)은 **AdSense 사이트 허용목록 + `ads.txt`**로 사실상 차단된다(§4-1).

## 6. 상업화 체크리스트

- [ ] 모든 에셋·폰트·라이브러리 **상업 라이선스** 확인·기록([licenses.md](./licenses.md))
- [ ] 호스팅이 상업/광고 **허용(ToS)** — 필요 시 이전
- [ ] **개인정보처리방침 + 동의(CMP)** 구비
- [ ] 광고가 **플레이를 끊지 않고** 경계에서만
- [ ] **오클릭·다크 패턴 없음**, 닫기 명확
- [ ] 광고 실패/차단 시에도 게임 정상
- [ ] **도박 오인 방지**(광고 카테고리·게임 표기)
- [ ] **게임 명칭 상표 검색**(KIPRIS — 통아저씨·악어 이빨 등)
- [ ] **게임물 등급분류** 대상/면제 여부 확인(국내)
