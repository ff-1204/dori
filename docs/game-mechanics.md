# 게임별 메커니즘 패턴

각 미니게임의 핵심 메커니즘·자료구조·구현 노트를 정리합니다.
공통 이론(난수·확률·공정성·FSM 등)은 [game-theory.md](./game-theory.md) 참고.

> **최우선 원칙**: 모든 구현은 [design-principles.md](./design-principles.md)(게이미피케이션·어포던스)를 우선한다.
> 조작은 눌러/돌려 보이게(시그니파이어), 화면과 결과는 일치(정직한 매핑), 피드백은 정직하게.

모든 게임은 공통 상태 흐름을 따른다: `IDLE → SETUP → BUILDUP → RESOLVE → RESULT → RESET`.

---

## A. 결정 돕기 게임

### A-1. 룰렛 (점심 메뉴)

- **핵심**: 원판을 돌려 포인터가 가리키는 칸이 결과.
- **자료구조**: 항목 리스트(+선택적 가중치).
- **판정**: 결과를 **먼저 정하고**(공정한 RNG), 그 각도에 멈추도록 회전 각을 역산 → 연출과 결과 일치.
  ```js
  const n = items.length;
  const winner = rng.between(0, n - 1);
  const sliceAngle = 360 / n;
  const target = 360 * spins - (winner * sliceAngle + sliceAngle / 2); // 여러 바퀴 + 목표칸
  this.tweens.add({ targets: wheel, angle: target, duration: 4000, ease: 'Cubic.easeOut' });
  ```
- **주스**: 감속(ease-out), 멈출 때 딸깍 사운드, 당첨 칸 확대/파티클.
- **가중치 변형**: 칸 크기를 가중치에 비례시키면 시각적으로도 확률이 드러남.

### A-2. 사다리타기

- **핵심**: 세로줄 N개 상단(참가자) → 하단(결과) 1:1 매핑. 가로 다리를 만나면 옆줄로 이동.
- **공정성**: 결과는 **순열(permutation)** → 전단사 자동 보장([game-theory.md](./game-theory.md) §3).
- **자료구조**: 가로 다리 목록 `rungs = [{row, col}]` (col과 col+1을 잇는다).
  - **제약**: 같은 행(row)에서 인접한 두 다리가 겹치면 안 됨(한 교차점에서 갈래 셋 방지).
  - 생성: 각 행마다 임의 col에 다리 추가하되, 바로 옆(col±1)에 이미 있으면 건너뜀.
- **경로 추적**: 위→아래로 내려가며 현재 행에 `col` 또는 `col-1`에 다리가 있으면 좌우로 한 칸 이동.
  ```js
  function trace(startCol, rungs, rows) {
    let col = startCol;
    for (let row = 0; row < rows; row++) {
      if (rungs.some(r => r.row === row && r.col === col))       col += 1;
      else if (rungs.some(r => r.row === row && r.col === col-1)) col -= 1;
    }
    return col; // 도착한 세로줄 인덱스
  }
  ```
- **주스**: 선을 따라 점이 이동하는 애니메이션(경로 하이라이트).
- **용도 프리셋**(트렌드 조사): 커피·벌칙·청소·당첨 — "1명 당첨, 나머지 통과" 패턴이 실사용의 대부분([trend-research.md](./trend-research.md) §1).
- **색상 연결**: 참가자 색 = 경로 색 = 결과 하이라이트 색(visual-polish §3-1a).

### A-3. 랜덤 핀볼

- **핵심**: 공을 떨어뜨려 물리 충돌로 여러 골 중 하나에 도착.
- **자료구조/엔진**: 물리 엔진 필요. Phaser **Arcade**(가벼움) 또는 **Matter**(정교한 반발/회전).
- **결정성 주의**: 물리는 프레임/부동소수 오차로 **비결정적**일 수 있음.
  - **정직한 매핑 원칙**: 결과를 몰래 조작(핀 배치/힘을 숨겨 유도)하지 않는다 → 화면과 결과 불일치는 어포던스 위반([design-principles.md](./design-principles.md)).
  - 재현성이 필요하면 **고정 시드 물리**로 투명하게 처리한다.
  - 순수 물리 랜덤이면: 골마다 확률이 균등한지 검증(플레이테스트/통계) 필요.
- **주스**: 핀 충돌음·튐, 카메라 흔들림, 골 도착 시 폭죽.
- **주도성(핵심)**: 플링코 바이럴의 심리는 "통제된 불확실성" — **낙하 지점을 플레이어가 선택**하게 해 주도성을 준다([trend-research.md](./trend-research.md) §4).

## B. 랜덤 뽑기 / 복불복 게임

### B-1. 러시안 룰렛

- **핵심**: 6칸 중 1칸에 "탄". 순서대로 방아쇠 → 걸리면 패배.
- **확률**: 순차 조건부. 첫 시도 1/6, 다음 1/5 … ([game-theory.md](./game-theory.md) §2).
- **자료구조**: 불리언 배열 또는 "탄 위치 인덱스" 하나 + 현재 시도 카운트.
  ```js
  const bulletSlot = rng.between(0, 5); // SETUP에서 1회 결정
  // 시도 t(0-base)에서: hit = (currentSlot === bulletSlot)
  ```
- **긴장 설계**: 시도마다 사운드 피치↑, 딜레이↑, 화면 점점 조임(§8 점진적 위험).

### B-2. 악어 이빨 누르기

- **핵심**: 여러 이빨 중 하나가 "트리거". 누르다가 트리거를 밟으면 입이 닫힘(패배).
- **자료구조**: 슬롯 배열 중 1개가 트리거(불리언 마스크). 누른 이빨은 비복원 제외.
  ```js
  const teeth = 12;
  const trap = rng.between(0, teeth - 1);   // SETUP에서 재배치
  const pressed = new Set();
  function press(i) {
    pressed.add(i);
    return i === trap; // true면 물림(패배)
  }
  ```
- **정직한 긴장 연출**: 트리거 여부에 맞춰 긴장을 쌓되, 안전한 칸을 위험한 것처럼 속이는 **가짜 연출은 지양**([design-principles.md](./design-principles.md)).
- **재배치**: 매 판 `trap`을 새로 뽑아 학습 방지.

### B-3. 해적통(구 통아저씨) (팝업 해적)

- **핵심**: 여러 칼 구멍 중 하나가 "발사 트리거". 순서대로 칼을 꽂다가 트리거에 꽂으면 인형이 튀어나옴.
- **구조**: 악어 이빨과 동일 패턴(슬롯 중 1개 트리거, 비복원 시도) + 튀어오름 연출.
  - 튀어오름: `Back.easeOut`/`Bounce`로 인형 팝업.
- **변형**: 트리거를 매 시도 확률로 결정(누적 위험 상승)하거나, SETUP에서 고정.

### B-4. 단청 (丹靑 — 양자택일 오라클)

- **핵심**: 질문 텍스트 → 붉(丹)/청(靑) 중 하나. **결정적 해시**라 서버 없이 모든 기기·브라우저에서 동일 답.
  ```js
  색 = FNV1a(`단청|${정규화(질문)}|${시간창}`) % 2;   // 0=붉, 1=청
  시간창 = floor((Date.now() + 9h) / 4h);            // KST 정각 경계(00/04/08/12/16/20)
  ```
- **정규화**: NFC → trim → 연속 공백 축약 → 소문자. "짜장면 "과 "짜장면"이 같은 답.
- **설계 포인트**:
  - **리롤 불가**: 같은 창 안에서는 몇 번을 물어도 같은 답 → "번복 불가"가 곧 긴장이자 공정성.
  - **글로벌 동기**: 순수 함수(입력+시간)이므로 저장·통신 없음 — 친구와 같은 질문을 각자 폰에서 물어도 같은 답.
  - **정직성**: "같은 질문 = 같은 답" 규칙을 안내 문구로 공개. **카운트다운은 표시하지 않음** — 초 단위 타이머는 도박적 긴급함(urgency)을 만들어 윤리 가드레일과 충돌.
  - 입력은 클라이언트에서만 소비(개인정보 이슈 없음, [commercial.md](./commercial.md) §2 부합).
- **해시**: FNV-1a 32bit — 퍼블릭 도메인 알고리즘을 스펙에서 직접 구현(서드파티 없음).

## C. 공통 설계 노트 (재사용 관점)

- **`MiniGame` 베이스 클래스 후보**: `setup()`, `buildup()`, `resolve()`, `showResult()`, `reset()` +
  공통 RNG/입력 잠금(`locked`)/시드 관리. 각 게임은 이를 상속.
- **결과-우선 vs 과정-우선**:
  - 결과를 먼저 정하고 연출로 보여주기(룰렛·사다리) → 공정성·검증 쉬움.
  - 과정(물리)에서 결과가 나오기(핀볼) → 자연스럽지만 균등성 검증 필요.
- **시드 관리**: 판마다 시드를 기록하면 "다시보기/분쟁 해결"이 가능([game-theory.md](./game-theory.md) §1, §4).
- **공정성 표시**: 트리거/당첨을 미리 정했더라도, 플레이어가 **결과가 조작되지 않았다고 느끼게** 연출(랜덤 배치가 보이게, 시드 노출 옵션).
