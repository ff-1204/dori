# 게임 개발 이론 — 결정/랜덤/복불복 게임

`dori`가 만들려는 게임들(점심 메뉴 룰렛, 사다리타기, 랜덤 핀볼, 러시안 룰렛, 악어 이빨, 통아저씨 등)은
모두 **난수·확률 기반의 짧은 파티 게임**입니다. 이 계열에 공통으로 쓰이는 이론을 정리합니다.
게임별 구체 구현은 [game-mechanics.md](./game-mechanics.md)를 참고하세요.

> **최우선 원칙**: 모든 설계는 [design-principles.md](./design-principles.md)(게이미피케이션·어포던스)를 우선한다.
> 아래 기법이 그와 충돌하면 그쪽을 따르고, 정직한 피드백·정직한 매핑·명확한 목표를 지킨다.

---

## 1. 난수와 RNG (Random Number Generation)

- **PRNG(의사난수)**: 컴퓨터 난수는 대부분 결정론적 알고리즘. 완전한 진짜 난수가 아니다.
- **시드(seed) 기반 난수**: 같은 시드 → 같은 결과열. **재현 가능성**은 공정성 분쟁 해결·리플레이·테스트에 중요.
- **`Math.random()`의 한계**: 시드 지정 불가, 품질/분포 보장 약함. 게임에선 엔진 RNG 사용 권장.
- **Phaser RNG**: `Phaser.Math.RND`는 시드 지정 가능.
  ```js
  const rng = new Phaser.Math.RandomDataGenerator(['dori-seed-2026']);
  rng.between(0, 5);          // 정수
  rng.pick(['짜장','짬뽕']);  // 배열에서 하나
  rng.shuffle(arr);           // 셔플(내부 Fisher-Yates)
  ```

## 2. 확률 기초

| 개념 | 설명 | 대표 게임 |
|------|------|-----------|
| **균등 분포(uniform)** | 모든 항목이 같은 확률 | 룰렛(균등), 통아저씨 |
| **가중치 랜덤(weighted)** | 항목마다 다른 확률 | 가중치 룰렛(칸 크기로 표시), 뽑기 등급 |
| **비복원 추출(without replacement)** | 뽑은 건 다시 안 나옴 | 사다리타기, 악어 이빨 |
| **복원 추출(with replacement)** | 매번 전체에서 다시 뽑음 | 반복 룰렛 |
| **조건부 확률** | 앞 결과가 뒤 확률을 바꿈 | 러시안 룰렛(1/6→1/5→…) |

**가중치 랜덤(누적 합 방식):**
```js
// weights: [{item:'A', w:3}, {item:'B', w:1}] → A가 3배 확률
function weightedPick(weights, rng) {
  const total = weights.reduce((s, x) => s + x.w, 0);
  let r = rng.frac() * total;      // 0 ~ total
  for (const x of weights) {
    if ((r -= x.w) < 0) return x.item;
  }
}
```

## 3. 셔플과 공정한 선택

- **Fisher-Yates 셔플**: 편향 없는 유일한 정석. 뒤에서부터 무작위 위치와 교환.
  ```js
  function shuffle(arr, rng) {
    for (let i = arr.length - 1; i > 0; i--) {
      const j = rng.between(0, i);
      [arr[i], arr[j]] = [arr[j], arr[i]];
    }
    return arr;
  }
  ```
- **안티패턴**: `sort(() => Math.random() - 0.5)` — 분포가 편향됨. 쓰지 말 것.
- **전단사(bijection) 보장**: 사다리타기처럼 "입력 N개 → 출력 N개가 1:1"이어야 하는 경우, 셔플은 순열(permutation)이라 자동으로 보장된다.

## 4. 공정성(Fairness)과 체감 공정성

- **진짜 랜덤 vs 체감 랜덤**: 진짜 랜덤은 종종 "몰림(streak)"을 만든다. 사람은 이를 불공정하게 느낀다.
- **도박사의 오류(gambler's fallacy)**: "계속 꽝이었으니 이번엔 당첨" — 독립 시행에선 틀린 직관. 설계자는 이 심리를 이해하되 **조작에 악용하지 않고** 정직한 범위에서만 완화한다.
- **몰림 완화 기법**: 비복원 추출, "최근 N회 제외" 등. (가중치를 몰래 바꾸는 **피티 시스템(pity system)**은 표시 확률과 실제 확률이 달라져 [design-principles.md](./design-principles.md) '정직한 확률'과 충돌 — 쓰려면 **확률을 공개하는 전제**에서만.)
- **투명성**: 시드 공개·결과 로그·검증 가능한 애니메이션(룰렛이 실제로 그 칸에 멈추는 게 보이게)으로 신뢰 확보.

## 5. 게임 필 / 주스(Juice)

짧은 랜덤 게임의 재미는 **결과 자체가 아니라 결과에 이르는 과정의 연출**에서 나온다.

- **기대감 → 긴장 → 해소(anticipation → tension → release)** 3박자.
- **피드백 채널**: 시각(애니메이션·파티클·플래시), 청각(효과음·BGM 고조), 햅틱(진동).
- **연출 기법**: 스쿼시&스트레치, 화면 흔들림(screen shake), 슬로모션, 줌인, 색 강조.
- **핵심 원칙**: 결과는 이미 정해져도, **연출로 긴장을 끌어야** 한다(룰렛은 결과보다 도는 시간이 재미).

## 6. 이징(Easing)과 애니메이션

- **이징 함수**: 시간에 따른 값 변화 곡선. Phaser Tween의 `ease` 옵션.
  - `Quad/Cubic.easeOut` — 룰렛·사다리 감속(빠르게 시작 → 천천히 멈춤).
  - `Back.easeOut` — 팝업·튀어나옴(살짝 넘쳤다 돌아옴).
  - `Elastic/Bounce` — 통통 튀는 마무리.
- **룰렛 감속 설계**: 각속도를 ease-out으로 줄여 "거의 멈출 듯 말 듯" 긴장을 만든다.
- **빌드업**: 복불복은 시도마다 간격을 늘리거나 사운드 피치를 올려 긴장을 누적.

## 7. 게임 루프와 상태 머신 (FSM, Finite State Machine)

이 계열 게임은 대부분 동일한 상태 흐름으로 추상화된다.

```
IDLE → SETUP → BUILDUP → RESOLVE → RESULT → (RESET → IDLE)
```

| 상태 | 하는 일 |
|------|---------|
| `IDLE` | 시작 대기, 입력 안내 |
| `SETUP` | 참가자/항목/시드 준비, 트리거 랜덤 배치 |
| `BUILDUP` | 연출·긴장 고조(룰렛 회전, 이빨 누르기 진행) |
| `RESOLVE` | 실제 결과 판정 |
| `RESULT` | 당첨/꽝 연출, 승패 표시 |
| `RESET` | 다시하기 |

- **입력 잠금**: `BUILDUP`/`RESOLVE` 동안 중복 입력을 막아 상태 꼬임 방지(`this.locked = true`).
- 상태별 로직을 분리하면 게임마다 재사용하기 좋다(공통 `MiniGame` 베이스 클래스 후보).

## 8. 긴장 설계 (복불복 특화)

- **점진적 위험 증가**: 러시안 룰렛처럼 남은 시도가 줄수록 확률↑ → 자연스러운 긴장 상승.
- **정직한 긴장 연출**: 결과에 이르는 과정을 늦추고 사운드/시각으로 긴장을 쌓는다. 단, 실제 상태와 다른 **가짜 연출(거짓 니어미스)로 오해를 주지 않는다** ([design-principles.md](./design-principles.md) 정직한 피드백).
- **무작위 트리거 배치**: 매 판 트리거 위치를 새로 섞어(비복원) 학습·꼼수를 방지.
- **되돌릴 수 없는 선택**: 한 번 누르면 확정 → 결정의 무게가 재미.

## 9. 동기와 보상 심리 (윤리 가드레일)

- **내재적 동기 우선**: 점수·배지 같은 외적 보상보다 자율성·유능감·재미(내재적 동기)를 우선한다([design-principles.md](./design-principles.md) SDT).
- **가변 비율 보상(variable ratio)**: 예측 불가한 보상은 몰입을 높인다. 단 **중독을 노린 다크 패턴은 금지**.
- **놀람과 반전**: 기대와 다른 결과가 감정 반응을 키운다.
- **공정한 벌칙**: 복불복의 벌칙은 가볍고 웃긴 쪽이 재플레이를 유도.

## 10. 데이터 구조

| 게임 | 핵심 자료구조 |
|------|--------------|
| 룰렛/뽑기 | 항목 리스트 + (선택)가중치 테이블 |
| 사다리타기 | 세로줄 배열 + 가로 다리(rung) 목록 = 그래프 |
| 핀볼 | 물리 바디(중력·반발계수·충돌) |
| 러시안 룰렛 | 원형 큐/불리언 배열 + 현재 인덱스 |
| 악어/통아저씨 | 슬롯 배열 중 1개가 트리거(불리언 마스크) |

## 11. 접근성 / UX

- **색각 이상 대응**: 색만으로 구분하지 말고 아이콘·라벨·패턴 병행.
- **결과 명확성**: 누가 걸렸는지/뭐가 당첨인지 크고 분명하게.
- **빠른 리플레이**: 결과 후 원터치로 다시 시작.
- **모바일 우선**: 큰 터치 타깃, 세로형 레이아웃([development.md](./development.md)의 720×1280 기준).
