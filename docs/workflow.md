# 개발 워크플로 — 이 프로젝트의 작업 절차

첫 세션(2026-07-16)에서 정착된 절차의 핵심 기록. 새 작업도 이 흐름을 따른다.

## 1. 기능 개발 파이프라인

```
조사 → 설계 → 구현 → 검산 → 문서화 → 커밋 → 배포 확인
```

1. **조사**: 트렌드·사용 맥락을 웹으로 검증하고 [trend-research.md](./trend-research.md)에 출처와 함께 기록. 지식으로 채운 값은 "검증 전"임을 인지하고 이후 데이터로 교정.
2. **설계**: [design-principles.md](./design-principles.md)(게이미피케이션·어포던스·정직함)를 통과하는지 먼저 판정. 충돌하는 기법은 시작 전에 제거.
3. **구현**: 디자인 토큰(`theme.js`)·공통 컴포넌트(`ui.js`·`MiniGame.js`)·표준 기법(색상 연결, Peak-End, 상대 좌표) 재사용.
4. **검산**: 브라우저 검증이 불가하므로 **요소 점유 좌표를 계산해 겹침 검사**(visual-polish 체크리스트). 레이아웃 총합이 1280을 넘는지도 계산.
5. **문서화**: 코드와 **같은 커밋**에서 관련 문서 갱신(game/development/…). 서드파티 추가 시 licenses.md **즉시** 기록.
6. **커밋**: Conventional Commits + 한국어, Co-Authored-By 없음, 커밋 전 상업 3문 점검([git.md](./git.md)).
7. **배포 확인**: 푸시 후 curl로 핵심 파일이 라이브에 반영됐는지 폴링(약 20–40초). 삭제한 파일은 404 확인.

## 2. 릴리스 절차

`/release` 커맨드(.claude/commands/release.md)로 자동화:
마지막 태그 이후 커밋 검토 → SemVer 결정 → CHANGELOG 작성 → README·메타·og 이미지 **숫자/문구 정합** → 커밋 → `vX.Y.Z` 주석 태그 → 배포 확인.

## 3. 이 프로젝트의 고정 규칙 (세션에서 확립)

- **정직함 최우선**: 화면=결과(정직한 매핑), 확률 공개(피티는 문서+화면 예고), 가짜 연출 금지, 카운트다운 같은 도박식 긴급함 금지.
- **색상 연결**: 결과·피드백 색 = 출처 요소의 색(visual-polish §3-1a). 참가자·칸·이빨 전부.
- **카피 톤**: "무료·설치 없이·지금 바로" 등 배너식 표현 금지 — 경험을 말한다(§3-5).
- **멀티 디바이스**: 상대 좌표, 터치 타깃 설계 88px+, hover는 보조.
- **상업 안전 상시**: 상업 허용 라이선스만·즉시 기록, 자산은 자체 제작 우선(이미지 = PowerShell System.Drawing, 소리 = Web Audio 합성, 아이콘 = 코드/이모지 텍스처).
- **시간대 시스템**: `timeOfDay.js` 단일 소스 — 분위기·인사말·메뉴·허브 라벨이 전부 여기서 파생(불일치 방지).

## 4. 배운 것 (실수 → 규칙)

| 사건 | 규칙화 |
|------|--------|
| 룰렛 포인터가 제목을 가림 | 요소 좌표 점유 범위 검산을 체크리스트化 |
| 허브 라벨 "점심"이 실제(저녁)와 불일치 | 파생 값은 공용 모듈에서만 계산 |
| 러시안 룰렛 발사 칸 오프바이원 | 회전 **후** 상태 기준으로 판정·표시 |
| "설치 없이 무료" 문구 지적 | 카피 톤 규칙 신설 |
| 8번째 게임 추가 시 레이아웃 초과 | 추가 전 총합 계산(1280 예산) |

> 세부 이력은 git log와 [CHANGELOG.md](../CHANGELOG.md) 참고. 이 문서는 "무엇을"이 아니라 **"어떻게"**를 기록한다.
