# 게임 기획/설계

`dori` 게임의 컨셉·현재 상태·로드맵을 정리한 문서입니다. 게임이 발전하면 이 문서에 계속 기록합니다.

## 컨셉

- 정체성: **결정 돕기 · 랜덤 뽑기 · 복불복**을 모두 포괄하는 캐주얼 미니게임 **컬렉션 사이트**
- 방향: 세로형(모바일 우선), 720×1280 기준, 화면에 맞춰 `FIT` 스케일
- 조작: 탭(터치/클릭) 중심의 가벼운 플레이
- 메뉴(허브)에서 게임을 골라 즐기는 **미니게임 포털** 형태

### 포괄 범위 & 수록 예정 게임

- **결정 돕기**: 점심 메뉴 룰렛 · 사다리타기 · 랜덤 핀볼
- **랜덤 뽑기 / 추첨**: 룰렛·사다리 등을 추첨 용도로 활용 + 뽑기 상자류 확장
- **복불복**: 러시안 룰렛 · 악어 이빨 누르기 · 통아저씨(팝업 해적)

> 세 범주 모두 난수·확률 기반이라 공통 이론([game-theory.md](./game-theory.md))과 메커니즘([game-mechanics.md](./game-mechanics.md))을 공유한다.

## 씬 구조

```
Boot → Preload → Hub → (게임 선택) → 각 미니게임 → (⬅ 메뉴) → Hub
```

| 씬 | 역할 |
|----|------|
| `Boot` | 최초 초기화 후 Preload로 전환 |
| `Preload` | 로딩 바 표시 후 Hub로 전환 |
| `Hub` | 타이틀 + 범주별 게임 선택(미구현은 '준비 중') |
| `Roulette` 등 | 각 미니게임(`MiniGame` 베이스 상속) |

## 현재 상태

- **허브**: 결정 돕기 · 랜덤 뽑기 · 복불복 범주로 게임 목록 노출, 미구현은 '준비 중'으로 흐리게(정직한 어포던스). 타이틀 "dori"는 그라디언트·그림자·강조선·은은한 호흡 연출, 하단에 제작자 GitHub 크레딧. 상단에 **QR 접속 모달(좌)**·**공유 버튼(우)**, 하단에 **효과음 토글(좌)·크레딧(중앙)·📲 바로가기(우, PWA 설치)**. 룰렛 버튼 라벨은 **시간대 식사(아침/점심/저녁/야식)로 동적 표기**(게임 내 동작과 일치).
- **메뉴 룰렛(구현 완료)**: **시간대별 메뉴 세트**(아침/점심/저녁)를 자동 선택. 트렌드 기반 기본 메뉴 + **사용자 추가/삭제**(localStorage 저장). 결과를 RNG로 먼저 정하고 포인터 칸에 정확히 멈추는 **정직한 매핑**, `Cubic.easeOut` 감속, 결과 팝업(색상 연결), 다시 돌리기.
- **사다리타기(구현 완료)**: 참가자 2~6명 + 결과 편집(localStorage), **용도 프리셋**(커피/벌칙/청소/당첨 — [trend-research.md](./trend-research.md)). '시작' 시 다리가 전부 공개되는 **정직한 매핑**, 이름 탭 → 참가자 색 경로 애니메이션 → 결과 하이라이트(**색상 연결**).
- **랜덤 핀볼(구현 완료)**: **낙하 지점 직접 선택**(주도성 — trend §4) + 실제 물리 낙하(Arcade). 도파민 설계: 핀 히트 플래시·튕김 카운터·**막판 슬로모+줌**·착지 **파티클 폭발**(Peak-End). 칸 라벨 편집(6칸 고정, localStorage).
- **뽑기 상자(구현 완료)**: **비복원 추첨**(Wheel of Names 인사이트 — trend §2). 뽑힌 항목은 화면에서 흐려져 남은 풀이 항상 보임(정직). 상자 흔들림 → 카드 팝(색상 연결). 항목 2~12개 편집 + 전부 되돌리기.
- **러시안 룰렛(구현 완료)**: 6약실 실린더, 순차 조건부 확률을 **정직하게 표시**("이번 확률 1/4"). 시도마다 서스펜스 증가, 빵!(위험색 Peak) — 벌칙은 유쾌하게 프레이밍.
- **악어 이빨(구현 완료)**: 12개 이빨 중 1개 함정(비복원). 눌림 즉각 피드백, 함정이면 위턱이 쾅(Peak). 가짜 덜컥임 없음(정직한 긴장).
- **통아저씨(구현 완료)**: 12개 칼 구멍 중 1개 트리거. 칼 꽂힘 애니메이션, 트리거면 아저씨 회전 발사(Back.easeOut Peak).
- **로또 번호 추첨(구현 완료)**: 1~45 중 6개 × 최대 5줄(줄 추가/빼기). **추첨은 매 정각 1번** — 뽑은 번호는 다음 정각까지 잠금·기록(localStorage). 복사·공유 버튼, 공식 로또 색 구간을 팔레트로 매핑(색상 연결). **"모든 조합의 확률은 같아요" 상시 고지**(허위·과장 방지), 재추첨 시각은 정적 표기(카운트다운 없음).
- **단청(丹靑, 구현 완료)**: 양자택일 오라클(붉을 단·푸를 청). 질문 입력 → **결정적 해시**(FNV-1a: "단청"+정규화 질문+4시간창)로 붉/청 확정. **서버 없이 어떤 기기에서 열어도 같은 질문 = 같은 답**, 변경은 KST 정각 4시간(00/04/08/12/16/20)에 1번 — 리롤 불가(운명성). "동일 답" 규칙은 안내 문구로 공개하되 **카운트다운은 표시하지 않음**(도박적 긴급함 배제).
- 공통 토큰(`theme.js`)·어포던스 버튼(`ui.js`)·FSM 베이스(`MiniGame.js`) 적용.

## 룰렛 수치 메모

- 시간대→식사: 아침(05–11) / 점심(11–17) / 저녁(17–21) / **야식(21–05)**. 각 세트 기본 10종(편집 가능, 2~16개)
- 저장 키: `dori.roulette.<breakfast|lunch|dinner|latenight>` (localStorage) — **시간대별 완전 독립**(서로 영향 없음)
- 초기화: 편집 패널의 **↺ 기본값** 버튼(현재 시간대만 복원) 또는 브라우저 **사이트 데이터 삭제**. Shift+F5는 HTTP 캐시만 지우므로 저장 메뉴에 영향 없음(브라우저 사양)
- 원판 반경 300px, 중심 `(360, 560)`, 회전 4바퀴 + 목표각, 3800ms, `Cubic.easeOut`
- 칸 색은 그리디 배정 — **인접 칸(순환 포함) 같은 색 금지**, 노랑은 감자튀김 전용. 라벨은 배경색으로 대비
- 기본 메뉴는 한국인 선호 음식 트렌드 기반(사용자가 자유롭게 편집)
- 🍟 **이스터에그**: `감자튀김`은 모든 시간대에 항상 존재(삭제 불가 — "감자튀김은 영원해요"), **노란 칸은 감자튀김 전용**(다른 칸은 노랑 제외 순환), 당첨 시 🍟 이모지 파티클 폭발(시스템 이모지를 렌더텍스처로 구움 — 외부 에셋 없음)
- 🍟 **보증(피티) 규칙 — 공개**: 메뉴 모드에서 **10번째 스핀은 감자튀김 확정**(`dori.roulette.spins` 카운터, 자연 당첨 시에도 리셋). game-theory §4의 "피티는 확률 공개 전제" 원칙에 따라 여기(공개 저장소 문서)에 명시한다. 숫자 모드에는 미적용
- 🍟 **보증 예고 힌트**: 카운터 9(다음이 보증)가 되면 감자튀김 칸이 **은은히 반짝이고 ✨ 표시** — 피티를 화면에서도 정직하게 예고
- 🍟 **숨은 치트**: 감자튀김 라벨 **더블탭** → 칸이 반짝인 뒤 감자튀김 확정 스핀(치트 사용 시에도 카운터 리셋)
- 🔢 **숫자 모드**: 우상단 토글로 메뉴 ↔ 숫자 룰렛 전환. 숫자 1~N(개수 2~16 설정, `dori.roulette.numberCount`), **감자튀김 없음**, 색은 노랑 포함 전체 순환. **모드는 저장하지 않음** — 허브에 갔다 오면 항상 기본(메뉴) 모드로 시작

## 신규 게임 수치 메모

- **핀볼**: 핀 7행(교차 7/6개), y 400~820, 공 r14/핀 r9, bounce 0.55, 중력 1500, 핀 히트 잔떨림 ±70, 슬로모 y>760(timeScale 2.1)+줌 1.06, 슬롯 6칸(라벨 편집, `dori.pinball.slots`, 기본값 `꽝·2등·1등·3등·2등·꽝`)
- **뽑기 상자**: 항목 2~12개(`dori.draw.items`), 비복원(뽑힌 인덱스 Set), 풀 항상 표시, 편집 시 기록 초기화(공정성)
- **러시안 룰렛**: 6약실, 시도별 서스펜스 300+130t ms, 확률 표기 1/(6−t). 발사 칸 표시는 회전 **후** 포인터에 온 약실 `(t+1)%6`
- **악어 이빨**: 12개(6×2), 함정 1, 터치 타깃 88×76, 위턱 세모 장식 없음, **함정 이빨은 빨간색으로 공개** / **통아저씨**: 12구멍(4×3), 타깃 84×88, 꽂힌 칼날은 마스크로 가려짐(옆 구멍 침범 없음), **트리거 구멍은 빨간색으로 공개**
- **로또**: 줄당 Fisher-Yates 셔플 후 앞 6개 오름차순, 잠금 창 = `floor(now/1h)`(모든 정수 오프셋 시간대에서 정각 경계), 저장 키 `dori.lotto`. 공 색: 1-10 노랑/11-20 파랑/21-30 빨강/31-40 회색/41-45 초록
- **허브**: 게임 목록은 **2열 그리드**(열 폭 304, 높이 80, 행 간 92) — 9개+ 확장 대응
- **단청**: `색 = FNV-1a("단청|" + NFC·trim·공백축약·소문자(질문) + "|" + floor((now+9h)/4h)) % 2` (0=붉 `#ff6b6b`, 1=청 `#6cc7ff`). 입력은 클라이언트에서만 처리(전송 없음). HTML 입력 오버레이(`dom.createContainer`), 질문 24자 제한

## 사다리타기 수치 메모

- 참가자 2~6명(모바일 가독성 한계), 이름/결과 4자 제한(라벨 겹침 방지)
- 다리 8행, 생성 확률 0.42/칸(같은 행 인접 금지), 경계마다 최소 1개 보장
- 저장 키: `dori.ladder.names` / `dori.ladder.results` (localStorage)
- 세로줄 x 90~630, 사다리 y 330~920, 경로 애니메이션 최대 2400ms `Sine.easeInOut`
- 참가자 색 6종(`PLAYER`) — 이름·경로·결과 하이라이트를 같은 색으로(색상 연결)
