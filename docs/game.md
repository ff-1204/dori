# 게임 기획/설계

`dori` 게임의 컨셉·현재 상태·로드맵을 정리한 문서입니다. 게임이 발전하면 이 문서에 계속 기록합니다.

## 컨셉

- 정체성: **결정 돕기 · 랜덤 뽑기 · 복불복**을 모두 포괄하는 캐주얼 미니게임 **컬렉션 사이트**
- 방향: 세로형(모바일 우선), 720×1280 기준, 화면에 맞춰 `FIT` 스케일
- 조작: 탭(터치/클릭) 중심의 가벼운 플레이
- 메뉴(허브)에서 게임을 골라 즐기는 **미니게임 포털** 형태

### 포괄 범위 & 수록 예정 게임

- **결정 돕기**: 점심 메뉴 룰렛 · 사다리타기 · 랜덤 핀볼
- **랜덤 뽑기 / 추첨**: 룰렛·사다리 등을 추첨 용도로 활용 + 뽑기 상자류 확장
- **복불복**: 러시안 룰렛 · 악어 이빨 누르기 · 통아저씨(팝업 해적)

> 세 범주 모두 난수·확률 기반이라 공통 이론([game-theory.md](./game-theory.md))과 메커니즘([game-mechanics.md](./game-mechanics.md))을 공유한다.

## 씬 구조

```
Boot → Preload → Menu → Game → (메뉴로) → Menu
```

| 씬 | 역할 |
|----|------|
| `Boot` | 최초 초기화 후 Preload로 전환 |
| `Preload` | 로딩 바 표시, 타일 텍스처 생성, Menu로 전환 |
| `Menu` | 타이틀 "dori" + 시작 버튼 |
| `Game` | 실제 플레이 |

## 현재 상태 (뼈대 데모)

- 5열 × 6행 타일 격자
- 타일을 탭하면 **점수 +10**, 팝 애니메이션 후 새 색으로 리스폰
- HUD: 점수 표시 + "메뉴" 버튼
- 타일 색상 5종을 코드로 생성(`tile0`~`tile4`)

> 아직 미니게임 규칙(승패/판정)은 없는 상태의 상호작용 데모입니다.
