# 게임 기획/설계

`dori` 게임의 컨셉·현재 상태·로드맵을 정리한 문서입니다. 게임이 발전하면 이 문서에 계속 기록합니다.

## 컨셉

- 정체성: **결정 돕기 · 랜덤 뽑기 · 복불복**을 모두 포괄하는 **결정 도구 모음 사이트** — 게임처럼 즐거운 연출·어포던스가 차별점
- **대외 포지셔닝 전환(2026-07-18)**: "미니게임 컬렉션" → "**게임처럼 즐거운 결정 도구**". 실질(승패·점수·목표 없음, 결정 보조)과 일치시키고 게임물 등급분류 논점을 줄이기 위함. 사용자 노출 카피·메타(JSON-LD `UtilitiesApplication`)·정적 페이지가 기준이며, **내부 문서·코드의 '게임' 용어는 그대로 둔다**(씬 키·변수·docs 명칭 변경 없음)
- 방향: 세로형(모바일 우선), 720×1280 기준, 화면에 맞춰 `FIT` 스케일
- 조작: 탭(터치/클릭) 중심의 가벼운 플레이
- 메뉴(허브)에서 게임을 골라 즐기는 **미니게임 포털** 형태

### 포괄 범위 & 수록 예정 게임

- **결정 돕기**: 점심 메뉴 룰렛 · 사다리타기 · 랜덤 핀볼 · 조 배정
- **랜덤 뽑기 / 추첨**: 룰렛·사다리 등을 추첨 용도로 활용 + 뽑기 상자류 확장
- **복불복**: 러시안 룰렛 · 악어 이빨 누르기 · 해적통(팝업 해적, 구 '통아저씨' — 상표 선제 회피 개명 2026-07-18)

> 세 범주 모두 난수·확률 기반이라 공통 이론([game-theory.md](./game-theory.md))과 메커니즘([game-mechanics.md](./game-mechanics.md))을 공유한다.

## 씬 구조

```
Boot → Preload → Hub → (게임 선택) → 각 미니게임 → (⬅ 메뉴) → Hub
```

| 씬 | 역할 |
|----|------|
| `Boot` | 최초 초기화 후 Preload로 전환 |
| `Preload` | 로딩 바 표시 후 Hub로 전환 |
| `Hub` | 타이틀 + 범주별 게임 선택(미구현은 '준비 중') |
| `Roulette` 등 | 각 미니게임(`MiniGame` 베이스 상속) |

## 현재 상태

- **허브**: 결정 돕기 · 랜덤 뽑기 · 복불복 범주로 게임 목록 노출, 미구현은 '준비 중'으로 흐리게(정직한 어포던스). 타이틀 "dori"(104px)는 그라디언트·그림자·강조선·은은한 호흡 연출, **시간대 인사말은 10종 × 5시간대 풀에서 해시 선택**(FNV-1a + KST 1시간 창 — 매시간 회전, 모두 같은 문구, `timeOfDay.js greetingForPhase`) 후 페이드 인, 게임 버튼은 순차 등장(스태거 40ms). 하단에 제작자 GitHub 크레딧. 상단에 **QR 접속 모달(좌)**·**🔗 주소 복사(우)**, 하단에 **효과음 토글(좌)·크레딧(중앙)·📲 바로가기(우, PWA 설치)**. 룰렛 버튼 라벨은 **시간대 식사(아침/점심/저녁/야식)로 동적 표기**(게임 내 동작과 일치).
- **메뉴 룰렛(구현 완료)**: **시간대별 메뉴 세트**(아침/점심/저녁)를 자동 선택. 트렌드 기반 기본 메뉴 + **사용자 추가/삭제**(localStorage 저장). 결과를 RNG로 먼저 정하고 포인터 칸에 정확히 멈추는 **정직한 매핑**, `Cubic.easeOut` 감속, **스핀 중 포인터가 지나는 칸을 그 칸 색으로 실시간 표기**(틱과 같은 리듬 — 포인터·텍스트·소리 일치), 결과 팝업(색상 연결), 다시 돌리기. **비복원 모드**('나온 건 제외' 토글): 나온 칸이 흐려지며 풀에서 제외, '제외 취소'로 복원. 편집에 **🗑 모두 지우기**(두 번 눌러 확정 — 감자튀김만 남김)·개별 삭제는 감자튀김 외 제한 없음, **칸 1개여도 스핀 가능**(칸이 1개뿐임이 화면에 그대로 보이므로 정직한 매핑 유지). 메뉴 추가는 **DOM 입력 오버레이**(조 배정 붙여넣기와 같은 패턴 — 테마 일치, Enter 확정) — `window.prompt` 대체. 숫자 모드는 **제거됨**(2026-07-21 — 결정 도구 포지셔닝에서 사용 빈도 낮음, 진입 버튼 삭제와 함께 기능째 정리).
- **사다리타기(구현 완료)**: 참가자 2–6명 + 결과 편집(localStorage), **결과 일괄 설정 버튼**('당첨' 1개 무작위 자리 + 나머지 '벌칙' — 용도별 프리셋 4종은 2026-07-21 단일 버튼으로 대체). 참가자 '+ 추가'는 **빈 'N번' 자동 추가**(입력창 없음), **이름·결과 수정은 시작 전 보드에서 더블탭**(시작 후엔 이름 탭 = 출발, 결과는 확정 단계라 수정 불가) — 수정 즉시 저장. **상태 사이클**: 설정 → 시작 → 진행 중 '다시 섞기'(즉시 새 사다리) → 모두 도착 시 버튼이 **'새 판'**으로 바뀌고 누르면 설정 단계 복귀(수정 가능 상태가 자동으로 열림, 2026-07-22). **👀 전체 결과**(시작 후에만, 판 아래 우측 — 편집 링크와 좌우 배치): 팝업으로 참가자 → 결과 전체 매핑을 참가자 색으로 표시(도착한 사람 ✓, 바깥 탭·닫기로 닫음) — 사다리가 전부 공개돼 있으므로 요약도 정직. 편집 모달은 **참가자 관리 + 결과 기본값 버튼만**(결과 개별 수정 칩은 보드 더블탭으로 이관, 2026-07-21). '시작' 시 다리가 전부 공개되는 **정직한 매핑**, 이름 탭 → 참가자 색 경로 애니메이션 → 결과 하이라이트(**색상 연결**). **레이아웃은 룰렛과 동일 패턴**(헤더 행 ⬅·제목 40px / 상단 문구 32px / 게임판 / 판 아래 ✎ 편집 26px / 주 버튼 y 1104) — **전 씬 공통 적용 완료(2026-07-22)**: 핀볼(🔀 섞기·✎ 편집 나란히)·뽑기 상자(✎ 편집·↺ 되돌리기)·조 배정(✎ 명단 편집·📋 복사)·로또(📋 복사·공유). 서브타이틀이 있는 씬(조 배정·러시안·악어·해적통·로또·단청)은 헤더 아래 **태그라인(y 128, 24px) + 동적 문구(y 190, 32px)** 구조. 단청은 답 공개 연출이 판 중심이라 결과 텍스트 위치는 유지(헤더·태그라인만 적용). **행 수는 인원 비례**(6+2×(n−1): 2명 8행–6명 16행) — 인접 교환 셔플의 섞임 편향(출발 열 근처 도착) 완화, **일괄 설정의 당첨 위치는 무작위 배치**(고정 자리면 특정 참가자가 구조적으로 유리해지는 문제 해소).
- **랜덤 핀볼(구현 완료)**: **낙하 지점 직접 선택**(주도성 — trend §4) + 실제 물리 낙하(Arcade). 도파민 설계: 핀 히트 플래시·튕김 카운터·**막판 슬로모+줌**(보드 전용 카메라 — UI·문구는 확대되지 않음, 2026-07-22)·착지 **파티클 폭발**(Peak-End). 칸 편집: **칸 수 4–10개**(칸이 많아지면 공·핀이 자동으로 작아짐, localStorage) — 룰렛 편집과 같은 문법: **칩 탭 = 그 칸 삭제**('− 칸 빼기' 제거), '＋ 칸 추가'는 '꽝' 추가, **이름은 보드 결과 칸 더블탭으로 수정**(2026-07-22). **🔀 맵 섞기**: 버튼으로 핀 배치 재생성 + 결과 칸 순서 셔플 — 전부 보이는 상태에서 바뀌므로 정직(낙하 전 확인 가능).
- **뽑기 상자(구현 완료)**: **비복원 추첨**(Wheel of Names 인사이트 — trend §2). 뽑힌 항목은 화면에서 흐려져 남은 풀이 항상 보임(정직). 상자 흔들림 → 카드 팝(색상 연결). 항목 2–12개 편집 + 전부 되돌리기. **'+ 추가'는 빈 'N번' 자동 추가, 이름은 풀 칩 더블탭으로 수정**(사다리와 같은 패턴 — 수정 시 뽑기 기록 초기화, 2026-07-22).
- **러시안 룰렛(구현 완료)**: 6약실 실린더, 순차 조건부 확률을 **정직하게 표시**("이번 확률 1/4"). 시도마다 서스펜스 증가, 빵!(위험색 Peak) — 벌칙은 유쾌하게 프레이밍.
- **악어 이빨(구현 완료)**: 12개 이빨 중 1개 함정(비복원). 눌림 즉각 피드백, 함정이면 위턱이 쾅(Peak — depth 정렬로 이빨이 입 안에 가려짐) → **잠시 물었다가 입을 다시 열며 빨간 함정 이빨 공개**(실물 완구 동작). 가짜 덜컥임 없음(정직한 긴장).
- **해적통(구현 완료)**: 12개 칼 구멍 중 1개 트리거. 칼 꽂힘 애니메이션, 트리거면 해적 회전 발사(Back.easeOut Peak) + **눈이 X로 바뀜(기절 연출)** — '다시 숨기기'로 복원.
- **로또 번호 추첨(구현 완료)**: 1–45 중 6개 × 최대 5줄(줄 추가/빼기). **추첨은 매 정각 1번** — 뽑은 번호는 다음 정각까지 잠금·기록(localStorage). 복사·공유 버튼, 공식 로또 색 구간을 팔레트로 매핑(색상 연결). **"모든 조합의 확률은 같아요" 상시 고지**(허위·과장 방지), 재추첨 시각은 정적 표기(카운트다운 없음).
- **조 배정(구현 완료)**: 번호(2–60) 또는 **이름 명단**으로 배정. **✎ 명단 편집(버퍼)**: 이름 추가(4자·최대 16명) 후 사람마다 탭으로 **참여 → 👑 조장 지정 → 쉬기(비참여)** 순환, ✕ 삭제 — 참여 2명 이상이면 자동으로 이름 모드. **📋 명단 복사 / 📥 붙여넣기**: 그룹 제목(조장/참여/쉼) 아래 이름을 나열하는 **순수 텍스트**(특수문자 없음)로 내보내고(빈 그룹도 제목 포함 — 빈 명단이면 채워 쓸 양식이 복사됨), 같은 형식을 DOM 텍스트영역 오버레이에 붙여넣어 교체(제목 없으면 전원 참여, 공백·쉼표·줄바꿈 구분 모두 허용 — 클립보드 왕복, 파일 없음). 제목 단어(조장·참여·쉼)는 이름으로 쓸 수 없음. **지정 조장은 셔플 후 조에 1명씩 우선 배치** — 조장이 조 수보다 **적어도 허용**(어느 조가 조장을 받을지도 셔플 — 앞 조 고정 방지), 초과분은 조원으로(규칙 공개). 지정이 없으면 '조장 뽑기' 토글(첫 도착 = 조장). Fisher-Yates 셔플 + 균형 분배(앞 조부터 1명 더), **칩 클러스터**(번호=원형·이름=알약형)로 통통 모여드는 표현(도착 순서 그대로 — 정직). 완료 시 조 인원 요약 + 조별 색 파티클, **📋 조 편성 복사**(결과 후에만 — 조장 👑 표시 + 딥링크).
- **단청(丹靑, 구현 완료)**: 양자택일 오라클(붉을 단·푸를 청). 질문 입력 → **결정적 해시**(FNV-1a: "단청"+정규화 질문+4시간창)로 붉/청 확정. **서버 없이 어떤 기기에서 열어도 같은 질문 = 같은 답**, 변경은 KST 정각 4시간(00/04/08/12/16/20)에 1번 — 리롤 불가(번복 없음). "동일 답" 규칙은 안내 문구로 공개하되 **카운트다운은 표시하지 않음**(도박적 긴급함 배제).
- 공통 토큰(`theme.js`)·어포던스 버튼(`ui.js`)·FSM 베이스(`MiniGame.js`) 적용.
- **뒤로가기 통합**(`nav.js`): 게임 진입 시 히스토리에 한 칸 쌓아 **모바일 백 제스처·브라우저 뒤로가기 = 허브 복귀**(사이트 이탈 아님), 허브에서는 평소처럼 나감. 인게임 ⬅ 버튼도 같은 히스토리 경로를 소비(스택-화면 일치), 브라우저 '앞으로'는 게임 재진입. **해시 딥링크** `#roulette` 등으로 특정 게임 바로 진입(새로고침 시에도 유지).
