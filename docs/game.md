# 게임 기획/설계

`dori` 게임의 컨셉·현재 상태·로드맵을 정리한 문서입니다. 게임이 발전하면 이 문서에 계속 기록합니다.

## 컨셉

- 장르: 2D 캐주얼 / 퍼즐
- 방향: 세로형(모바일 우선), 720×1280 기준, 화면에 맞춰 `FIT` 스케일
- 조작: 탭(터치/클릭) 중심의 가벼운 플레이

## 씬 구조

```
Boot → Preload → Menu → Game → (메뉴로) → Menu
```

| 씬 | 역할 |
|----|------|
| `Boot` | 최초 초기화 후 Preload로 전환 |
| `Preload` | 로딩 바 표시, 타일 텍스처 생성, Menu로 전환 |
| `Menu` | 타이틀 "dori" + 시작 버튼 |
| `Game` | 실제 플레이 |

## 현재 상태 (뼈대 데모)

- 5열 × 6행 타일 격자
- 타일을 탭하면 **점수 +10**, 팝 애니메이션 후 새 색으로 리스폰
- HUD: 점수 표시 + "메뉴" 버튼
- 타일 색상 5종을 코드로 생성(`tile0`~`tile4`)

> 아직 퍼즐 규칙(매칭/승패)은 없는 상태의 상호작용 데모입니다.

## 로드맵

- [ ] 매치-3 로직: 인접 타일 스왑 → 3개 이상 매칭 검사 → 제거 → 낙하 → 연쇄(콤보)
- [ ] 유효하지 않은 스왑 되돌리기
- [ ] 스테이지/목표(점수, 이동 횟수 제한)
- [ ] 사운드 & 파티클 이펙트
- [ ] 최고 점수 저장(`localStorage`)
- [ ] 실제 아트 에셋 교체(현재는 코드 생성 텍스처)

## 밸런스/수치 메모

- 보드 크기: `cols = 5`, `rows = 6`
- 타일 크기: 96px, 간격 12px
- 팝 점수: 10점 (매치-3 도입 시 매칭 수/콤보에 따라 조정 예정)
